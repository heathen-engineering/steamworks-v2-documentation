https://partner.steamgames.com/doc/api/ISteamInventory#ConsumeItem

Consumes items from a user's inventory. If the quantity of the given item goes to zero, it is permanently removed.

Once an item is removed it cannot be recovered. This is not for the faint of heart - if your game implements item removal at all, a high-friction UI confirmation process is highly recommended. ConsumeItem can be restricted to certain item definitions or fully blocked via the Steamworks website to minimize support/abuse issues such as the classic "my brother borrowed my laptop and deleted all of my rare items".

Parameters
instanceIdThe instance id of the item to consume
callbackThe callback to invoke when the request has been completed by Steamworks Inventory Services
Returns
True if the request was sent, false otherwise.

static void HeathenEngineering.SteamAPI.SteamworksInventoryTools.StartPurchase ( IEnumerable< SteamItemDef_t >  items,
IEnumerable< uint >  itemQuantity,
Action< SteamInventoryStartPurchaseResult_t, bool >  callback = null 
)
static

Starts the purchase process for the user

Starts the purchase process for the user, given a "shopping cart" of item definitions that the user would like to buy. The user will be prompted in the Steamworks Overlay to complete the purchase in their local currency, funding their Steamworks Wallet if necessary, etc.

If the purchase process was started successfully, then m_ulOrderID and m_ulTransID will be valid in the SteamInventoryStartPurchaseResult_t callback.

Parameters
itemsA collection of the items to be purchased
itemQuantityA collection of the quantities of each item to be purchased ... length must match length of items
callback[Optional] returns the order ID and transaction ID if the purchase was started successfuly otherwise it contains the EResult code whcih may indicate the reason for failure.
