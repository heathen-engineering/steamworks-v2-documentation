Detailed Description

This replaces the SteamManager concept from classic Steamworks.NET

The SteamworksClientApiSystem initalizes the client SteamAPI and handles callbacks for the system. For the convenance of users using a singleton model this class also provides a Instance static member and wraps all major funcitons and event of the SteamworksClientApiSettings object.

The SteamworksClientApiSystem is the core compoenent to the Heathen Steamworks kit and replaces the funcitonality present in Steamworks.NET's SteamManager. The primary funciton of the manager is to operate the update loop required by the Steamworks API and to handle and direct callbacks from the Steamworks API.

It is strongly advised that you never unload or reload the SteamworksClientApiSystem for example you should not place the SteamworksClientApiSystem in your title scene because that scene will be unloaded and reloaded multiple times. Even if you mark the object as Do Not Destroy on Load, on reload of the title or similar scene Unity will create a second SteamworksClientApiSystem creating issues with the memory it manages.

The recomended approch is place your SteamworksClientApiSystem and any other "manager" in a bootstrap scene that loads first and is never reloaded through the life of the game. This will help insure that through the life of your users play session 1 and exsactly 1 SteamworksClientApiSystem is created and never destroyed. While there are other approches you could take to insure this using a simple bootstrap scene is typically the simplest. For more information on this subject see .
