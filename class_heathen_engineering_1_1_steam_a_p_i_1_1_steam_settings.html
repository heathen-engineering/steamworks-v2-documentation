Detailed Description

The root of Heathen Engieering's Steamworks system. SteamworksClientApiSettings provides access to all core funcitonality including stats, achievements, the friend system and the overlay system.

The SteamworksClientApiSettings object is the root of Heathen Engineering's Steamworks kit. SteamworksClientApiSettings contains the configuration for the fundamental systems of the Steamworks API and provides access to all core funcitonality. You can easily access the active SteamworksClientApiSettings object any time via current a static member that is populated on initalization of the Steamworks API with the settings that are being used to configure it.

SteamworksClientApiSettings is divided into 2 major areas being client and server. The client member provides easy access to features and systems relivent for your "client" that is the applicaiton the end user is actually playing e.g. your game. This would include features such as overlay, friends, clans, stats, achievements, etc. server in contrast deals with tthe configuraiton of Steamworks Game Server features and only comes into play for server builds. Note that the server member and its funcitonality are stripped out of client builds, that is it is only accessable in a server build and in the Unity Editor

◆ achievements

List<AchievementObject> HeathenEngineering.SteamAPI.SteamSettings.achievements = new List<AchievementObject>()

The registered achievements assoceated with this configuration

This collection is used by Valve callbacks to match incoming stat updates and records the value against the approprete achievement. Put more simply if a stat is listed here then the system will update that SteamAchievementData object with state changes as that information comes in from the Valve backend insuring that these AchievementObject objects are an up to date snap shot of the local user's achievement value. For servers these objects simplify fetching and settting stat values for targeted users but of course dosn't cashe values for a local user since server's have no local user.

◆ ApplicationId

AppId_t HeathenEngineering.SteamAPI.SteamSettings.ApplicationId
staticget

The AppId_t value configured and initalized for.

This is the app id value the developer would have typed in to the Unity Editor when setting up the project. Note that hackers can easily modify this value to cause the Steamworks API to initalize as a different game or can use the steam_appid.txt to force the Steamworks API to register as a different ID. You can confirm what ID Valve sees this program as running as by calling Utilities.GetAppId you can then compare this fixed value to insure your user is not attempting to manipulate your program. In addition if you are integrating deeply with the Steamworks API such as using stats, achievements, leaderboards and other features with a configuration specific to your app ID ... this will further insure that if a user manages to initalize as an app other than your App ID ... such as an attempt to pirate your game that these features will break insuring a degraded experance for pirates.

◆ Server

GameServer HeathenEngineering.SteamAPI.SteamSettings.Server
staticget

Static access to the active GameServer object.

Note that when using CONDITIONAL_COMPILE script define this will be striped out of client builds.

The GameServer object provides easy access to Steamworks Game Server configuraiton and server only features. Note that your server can be a Steamworks Game Server and not have to use the Steamworks Networking transports ... e.g. you can use any transport you like and host anywhere you like. Being a Steamworks Game Server simply means that your server has initalized the Steamworks API and registered its self against Valve's backend ... in addition if this server has an IP address of a trusted server as defined in your app configuration on the Steamworks Portal, then it may perform GS only actions such as setting stats and achievments that are marked as GS only.
